# セキュリティポリシー

## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、公開 Issue ではなく、以下の方法で報告してください。

- メール: （セキュリティ連絡先を設定してください）
- GitHub Security Advisory: リポジトリの Security タブから報告

## 対応方針

- 報告受領後 48 時間以内に初回応答
- 重大な脆弱性は最優先で修正
- 修正完了後にセキュリティアドバイザリを公開

## セキュリティ設計方針

- 認証: Entra ID + HENNGE ONE による SSO/MFA 必須
- 通信: HTTPS（TLS 1.2+）のみ
- データ保存: AES-256 暗号化
- 監査ログ: 改ざん防止（追記専用＋ハッシュチェーン）
- 依存関係: GPL系ライブラリ不使用、Dependabot による自動更新
