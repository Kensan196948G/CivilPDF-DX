## 1. 目的と背景

### 1.1 建設業PDF業務の現状課題

建設・土木業界では、業務の根幹をなす図面・施工記録・安全書類・契約書類のほぼ全てがPDFで流通している。しかし、現場のPDF管理は以下の構造的な課題を抱えている。

| 課題カテゴリ | 具体的な問題 | 業務への影響 |
|---|---|---|
| **バージョン管理の不在** | 図面改訂版が現場・事務所・本社で別々に保存され、どれが最新か判別できない | 旧版図面による施工ミス・手戻り |
| **承認フローの非効率** | 押印のためだけに現場→事務所→本社を書類が往来する | 承認待ちによる工期遅延（平均2〜5日/件） |
| **電子納品の属人化** | 国土交通省電子納品要領への適合チェックが担当者の経験と勘に依存する | 納品リジェクト・再提出コスト |
| **検索不能なPDF群** | 手書きメモ・スキャンPDFがテキスト化されず、必要書類の特定に時間を要する | 1件あたり平均15〜30分の探索コスト |
| **セキュリティ統制の欠如** | 誰がどの図面を閲覧・コピーしたか追跡できない。USBへの抜き取りを防止できない | 情報漏洩リスク・コンプライアンス違反 |
| **デジタル・紙の混在** | 電子承認と紙押印が混在し、証拠性・法的有効性が曖昧 | 紙保管コスト・文書の二重管理 |

### 1.2 解決方針

CivilPDF-DX は上記課題を以下のアプローチで解決する。

```
【課題】バージョン管理の不在
  → 【解決】文書バージョン管理 + クラウド単一ソース + 変更差分比較

【課題】承認フローの非効率
  → 【解決】多段階電子承認ワークフロー + 電子印鑑 + モバイル対応

【課題】電子納品の属人化
  → 【解決】国交省要領準拠の自動チェッカー + PDF/A自動変換

【課題】検索不能なPDF
  → 【解決】OCR（日本語手書き対応）+ AIセマンティック検索

【課題】セキュリティ統制の欠如
  → 【解決】全操作の監査ログ + RBAC + デバイス制御 + Entra ID統合

【課題】デジタル・紙の混在
  → 【解決】PAdES-LTV長期電子署名 + 完全ペーパーレスワークフロー
```

### 1.3 プロジェクトスコープ

| スコープ | 内容 |
|---|---|
| **対象業界** | 建設業・土木業（ゼネコン・サブコン・建設コンサルタント） |
| **対象規模** | 従業員50〜5,000名の中堅〜大手企業 |
| **対象地域** | 日本国内（日本語UI・国交省要領準拠） |
| **スコープ外** | 海外展開・BIM 3D処理・受発注EDI連携（Phase 3以降） |

## 2. ターゲットユーザー

### 2.1 ユーザーペルソナと役割

| ロール（システム） | 職種 | 主な業務 | 利用モジュール |
|---|---|---|---|
| **admin** | システム管理者・IT担当 | ユーザー管理・システム設定・監査 | 管理コンソール WebUI |
| **manager** | 現場監督・PMO・品質管理責任者 | 承認・押印・プロジェクト統括 | GUIアプリ + WebUI |
| **engineer** | 現場エンジニア・設計担当 | 図面作成・赤入れ・施工記録 | GUIアプリ |
| **viewer** | 施主・発注者・監理者 | 閲覧・コメント・確認 | GUIアプリ（読み取り専用） |

### 2.2 ユーザー詳細プロファイル

#### 現場エンジニア（engineer）

- **業務課題:** タブレット・スマートフォンでの図面参照、手書きメモのデジタル化、赤入れ後の上司への提出
- **技術リテラシー:** 中程度（CADツール経験あり、クラウドツール経験は限定的）
- **利用環境:** 現場（屋外・ネット不安定）/ オフライン環境でも動作が必須
- **主な操作:** PDF閲覧・注釈・赤入れ・写真台帳作成・OCR・電子印鑑
- **要求:** 直感的UI、オフライン対応、高速PDF表示（大容量図面A1対応）

#### 現場監督・PMO（manager）

- **業務課題:** 複数プロジェクトの進捗把握、承認ボトルネックの解消、品質記録の確保
- **技術リテラシー:** 中〜高程度
- **利用環境:** 事務所PC + 現場タブレット
- **主な操作:** 承認ワークフロー管理・図面比較・是正指示書発行・電子押印
- **要求:** 承認ステータスの一元管理、差分比較の視覚化

#### 管理者（admin）

- **業務課題:** ユーザー権限管理・コンプライアンス対応・監査証跡の提出
- **技術リテラシー:** 高（IT担当）
- **利用環境:** 事務所PC・管理コンソール専用
- **主な操作:** ユーザーCRUD・ロール管理・監査ログ確認・SSO設定・AI利用ポリシー制御
- **要求:** 詳細な権限制御、全操作の監査ログ、セキュリティアラート

#### 閲覧者・施主（viewer）

- **業務課題:** 進捗確認・竣工書類の受領・コメント提出
- **技術リテラシー:** 低〜中（PCリテラシーは様々）
- **利用環境:** Webブラウザ（インストール不要を希望）
- **主な操作:** PDF閲覧・コメント・確認印
- **要求:** インストール不要・シンプルなUI・日本語サポート

## 付録

### A. 用語定義

| 用語 | 定義 |
|---|---|
| 電子納品 | 建設工事の成果品をデジタルデータで発注者（国土交通省等）に納品する仕組み |
| PDF/A | PDFの長期保存のための国際標準（ISO 19005）。フォント全埋め込み・外部参照禁止等の制約がある |
| PAdES | PDF高度電子署名形式（ETSI規格）。長期署名（LTV）はタイムスタンプを用いて署名の有効性を長期的に維持する |
| 是正指示書 | 施工不具合に対して現場監督が発行する指示文書 |
| KY活動 | 危険予知活動。工事開始前に作業の危険ポイントを洗い出す安全活動 |
| RBAC | Role-Based Access Control。ユーザーのロールに基づいてアクセス権を制御する仕組み |
| SSO | Single Sign-On。複数のサービスに一度の認証でアクセスできる仕組み |
| OCR | Optical Character Recognition。PDF/画像内の文字をテキストデータに変換する技術 |
