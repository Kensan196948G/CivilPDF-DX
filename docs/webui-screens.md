## 実装済み画面（MVP）

### 1. ログイン画面

**URL:** `/login`
**ファイル:** `pages/Login.tsx`
**実装状況:** 完成

#### 目的
ユーザー認証を行い、認証済みユーザーをダッシュボードへ誘導する。
認証不要のルートはこの画面のみ。`ProtectedRoute` により未認証ユーザーはここへリダイレクトされる。

#### 主要コンポーネント

| コンポーネント | 説明 |
|---|---|
| ロゴ・システム名 | 「CivilPDF DX」タイトル + サブタイトル（「建設業向け図面・書類管理システム」） |
| メールアドレス入力 | `<input type="email">` — HTML5 バリデーション付き |
| パスワード入力 | `<input type="password">` |
| エラーメッセージ | 認証失敗時に表示（「メールアドレスまたはパスワードが正しくありません」） |
| ログインボタン | 送信中は `disabled` + 「ログイン中...」ラベルに変化 |

#### API エンドポイント

| メソッド | パス | 説明 |
|---|---|---|
| `POST` | `/api/v1/auth/token` | `application/x-www-form-urlencoded` でメール + パスワード送信 → `{access_token, refresh_token}` |
| `GET` | `/api/v1/auth/me` | トークン取得後にユーザー情報取得 → Zustand store へ保存 |

#### 認証フロー
1. フォーム送信 → `POST /api/v1/auth/token`
2. `access_token` / `refresh_token` を `localStorage` に保存
3. `GET /api/v1/auth/me` でユーザー情報取得
4. Zustand `useAuthStore.setUser()` にユーザーをセット
5. `navigate('/dashboard')` でリダイレクト

#### 権限
認証不要（パブリックルート）。

### 2. ダッシュボード

**URL:** `/dashboard`
**ファイル:** `pages/Dashboard.tsx`
**実装状況:** 完成

#### 目的
ログイン後のトップ画面。プロジェクト・ドキュメント・ワークフローの統計を一目で把握し、最近のドキュメントへ素早くアクセスできる。

#### 主要コンポーネント

| コンポーネント | 説明 |
|---|---|
| ウェルカムメッセージ | 「ようこそ、{full_name / email} さん」 |
| StatCard × 4 | ドキュメント総数・プロジェクト数・ワークフロー総数・承認待ち件数を大きな数字で表示。それぞれ青・緑・紫・オレンジのアクセントカラー |
| 最近のドキュメントテーブル | 最新 5 件。タイトル・種別・ステータス（バッジ）・登録日を表示 |

#### API エンドポイント

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/documents` | 全ドキュメント取得（`listDocuments()`） |
| `GET` | `/api/v1/projects` | 全プロジェクト取得（`listProjects()`） |
| `GET` | `/api/v1/workflows` | 全ワークフロー取得（`listWorkflows()`） |

TanStack Query の `useQuery` で並列取得。キャッシュキー: `['documents']`, `['projects']`, `['workflows']`。

#### 権限
全ロールでアクセス可（`ProtectedRoute` により認証必須）。

### 3. ドキュメント管理

**URL:** `/documents`
**ファイル:** `pages/Documents.tsx`
**実装状況:** 完成

#### 目的
PDF ファイルのアップロード・一覧表示・削除を行うメイン画面。建設業で扱う図面・仕様書・報告書・契約書などの文書を一元管理する。

#### 主要コンポーネント

| コンポーネント | 説明 |
|---|---|
| ページヘッダー | 「ドキュメント」タイトル + 「+ アップロード」ボタン |
| アップロードフォーム | 「アップロード」ボタンクリックで展開。タイトル・プロジェクト選択・種別選択・PDF ファイル選択の 4 フィールド |
| 種別セレクト | 図面 / 仕様書 / 報告書 / 契約書 / その他（`DocumentType` enum に対応） |
| ドキュメントテーブル | タイトル・種別・ステータスバッジ・ファイルサイズ (KB)・登録日・削除ボタンの列 |
| ローディング状態 | `isLoading` 時は「読み込み中...」テキスト表示 |
| 空状態 | 「ドキュメントがありません」テキスト表示 |

#### API エンドポイント

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/documents` | ドキュメント一覧取得 |
| `POST` | `/api/v1/documents` | PDF アップロード（`multipart/form-data`: `project_id`, `title`, `document_type`, `file`） |
| `DELETE` | `/api/v1/documents/{id}` | ドキュメント削除 |

TanStack Query `useMutation` でアップロード・削除操作。成功後に `['documents']` クエリを無効化して再取得。

#### ステータス値

| ステータス値 | 表示 |
|---|---|
| `draft` | 下書き |
| `pending_review` | 承認待ち |
| `approved` | 承認済み |
| `rejected` | 却下 |
| `archived` | アーカイブ |

#### 権限

| ロール | 操作 |
|---|---|
| viewer | 一覧閲覧のみ |
| engineer | アップロード・自身の文書の削除 |
| manager / admin | アップロード・全文書の削除 |

### 5. 承認ワークフロー

**URL:** `/workflows`
**ファイル:** `pages/Workflows.tsx`
**実装状況:** 完成

#### 目的
ドキュメントの承認フローを可視化する。各ワークフローのステータスと承認ステップの進捗を一覧表示する。施工図・竣工図などの承認回付の状況把握に使用する。

#### 主要コンポーネント

| コンポーネント | 説明 |
|---|---|
| ページヘッダー | 「承認ワークフロー」タイトル |
| ワークフローカード | 各ワークフローをカード形式で表示。タイトル + 全体ステータスバッジ |
| ステップ一覧 | カード内にステップを横並びで表示。ステップ番号・ステップ名・ステータスバッジ・コメントを表示 |
| 作成日表示 | カード下部に `created_at` を日本語ロケールで表示 |
| ローディング状態 | 「読み込み中...」表示 |
| 空状態 | 「ワークフローがありません」表示 |

#### ステータスバッジ定義

| ステータス値 | 日本語ラベル | スタイル |
|---|---|---|
| `pending` | 待機中 | グレー |
| `in_progress` | 審査中 | オレンジ |
| `approved` | 承認済 | グリーン |
| `rejected` | 却下 | レッド |

#### API エンドポイント

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/workflows` | ワークフロー一覧取得（`listWorkflows()`） |
| `GET` | `/api/v1/workflows/{id}` | 個別ワークフロー取得 |
| `POST` | `/api/v1/workflows` | ワークフロー作成（`{document_id, approver_ids[]}`） |
| `POST` | `/api/v1/workflows/{id}/steps/{sid}/decide` | 承認/却下決定（`{decision: "approve"\|"reject", comment?}`） |

フロントエンド API クライアント（`api/workflows.ts`）では `approveStep()` / `rejectStep()` として個別ラップ済み。

#### 権限

| ロール | 操作 |
|---|---|
| viewer | 一覧閲覧のみ |
| engineer | 自分が承認者のステップへの決定操作 |
| manager | 自身の担当ステップ操作 + ワークフロー作成 |
| admin | 全操作 |

## 認証ストア

Zustand の `useAuthStore`（`src/console/frontend/src/store/auth.ts`）。

| フィールド / アクション | 型 | 説明 |
|---|---|---|
| `user` | `UserResponse \| null` | ログイン中のユーザー情報 |
| `setUser(user)` | `(user: UserResponse) => void` | ユーザー情報をセット |
| `clearUser()` | `() => void` | ログアウト時にクリア |
