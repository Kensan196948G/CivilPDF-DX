## 1. メイン画面 / ファイルブラウザ

### 目的
プロジェクト・フォルダ・文書のツリー構造を一覧表示し、PDF Viewer や各ツールへの起点となる画面。アプリ起動後に最初に表示される。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 左ペイン: プロジェクトツリー | プロジェクト → フォルダ → ファイルの 3 階層ツリー。折りたたみ可能 |
| 右ペイン: ファイル一覧 | サムネイル表示 / リスト表示 切替。タイトル・種別・ステータス・更新日・サイズを表示 |
| ツールバー | 新規プロジェクト・フォルダ・ファイルのアップロードボタン |
| 検索バー | ファイル名・タグ・OCR テキストのインクリメンタル検索 |
| ステータスバー | 同期状態・接続先サーバー・ログインユーザー表示 |
| コンテキストメニュー | 右クリック → 開く・比較・削除・承認申請・エクスポート |
| フィルターパネル | 文書種別・ステータス・日付範囲によるフィルタリング |

### 主要アクション

- ファイルダブルクリック → PDF Viewer を開く
- ファイル右クリック → 比較対象に追加（図面比較ビューアへ）
- ドラッグ＆ドロップ → PDF ファイルのアップロード
- プロジェクト選択 → Backend API からドキュメント一覧を取得して表示
- 「承認申請」メニュー → 承認ワークフロー作成ダイアログを表示

### 権限

| ロール | 操作可能範囲 |
|---|---|
| viewer | 閲覧・検索のみ |
| engineer | アップロード・削除（自分の文書）・承認申請 |
| manager | 全ドキュメントの操作・フォルダ管理 |
| admin | 全操作・プロジェクト管理 |

## 2. PDF Viewer

### 目的
PDF ファイルを高品質で表示する。図面・仕様書・契約書など全文書種別の閲覧基本画面。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| PDF レンダリングエリア | 高解像度 PDF 表示（DPI 設定可）。タッチ・マウスホイールズーム対応 |
| ページナビゲーション | 前/次ページ・ページ番号入力・サムネイル一覧サイドバー |
| ズームコントロール | 25%〜800%、フィットページ・フィット幅・実寸ボタン |
| 回転ボタン | 90° 単位の回転（表示のみ、ファイル変更なし） |
| 印刷ボタン | 印刷ダイアログ（印刷範囲・部数・向き指定） |
| 注釈レイヤー表示切替 | 既存注釈の表示 / 非表示トグル |
| OCR テキストオーバーレイ | スキャン PDF に対して OCR テキストを半透明オーバーレイ表示 |
| AI 要約パネル | 右サイドバーに現在ページの AI 要約を表示（将来実装） |
| ドキュメント情報パネル | ファイル名・ページ数・PDF/A 準拠・タグ・登録者・ステータス表示 |

### 主要アクション

- マウスホイール / ピンチ → ズームイン・アウト
- スペースキー / 矢印キー → ページスクロール
- 「注釈を追加」ボタン → 注釈・赤入れエディタへ切替
- 「OCR 表示」ボタン → OCR 結果ビューアへ切替
- 「比較」ボタン → 図面比較ビューアで本文書を左ペインに開く
- テキスト選択 → クリップボードコピー（OCR テキストのみ）

### 権限
全ロールで閲覧可。注釈追加は engineer 以上。

## 3. 注釈・赤入れエディタ

### 目的
PDF に対してフリーハンド描画・テキスト注釈・図形・スタンプ注釈を追加し、設計変更指示・検査指摘事項を記録する。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 描画ツールバー | ペン（フリーハンド）・直線・矩形・楕円・矢印ツール |
| テキストツール | テキストボックス追加。フォント・サイズ・色設定 |
| 強調ツール | ハイライト・下線・取消線（OCR テキストレイヤー対応） |
| スタンプツール | 「承認」「否認」「確認」「参照」などの定型スタンプ選択 |
| カラーパレット | 赤・青・緑・黄・黒など注釈色選択 |
| 線幅スライダー | 0.5pt〜10pt の線幅調整 |
| 注釈一覧サイドバー | 全注釈のリスト表示・クリックでジャンプ・フィルタリング |
| 元に戻す / やり直し | Ctrl+Z / Ctrl+Y |
| 保存ボタン | 注釈を Backend API へ保存（FDF または注釈 JSON 形式） |
| エクスポートボタン | 注釈フラット化 PDF のエクスポート |

### 主要アクション

- ペンツール選択 → マウスドラッグで自由描画
- テキストツール → クリックでテキストボックス配置・ダブルクリックで編集
- スタンプ選択 → クリックで任意位置に配置・ドラッグで位置調整
- 注釈クリック → プロパティパネルで色・サイズ・作成者・コメント編集
- 「保存」→ `PATCH /api/v1/documents/{id}` で注釈データを保存

### 権限

| ロール | 操作 |
|---|---|
| viewer | 閲覧のみ（既存注釈表示） |
| engineer | 自分の注釈の追加・編集・削除 |
| manager | 全注釈の操作 |
| admin | 全操作 + 注釈ロック設定 |

## 4. 図面比較ビューア

### 目的
同一図面の 2 バージョンを左右に並列表示し、変更箇所を自動検出・ハイライトする。設計変更の確認・照査作業を効率化する。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 左ペイン | 旧バージョン表示（ファイル名・バージョン番号・日付表示） |
| 右ペイン | 新バージョン表示 |
| 差分オーバーレイ | 追加箇所（緑）・削除箇所（赤）・変更箇所（黄）をハイライト |
| 同期スクロール | 左右同時スクロール（解除可能） |
| 差分ナビゲーション | 「前の差分」「次の差分」ボタンでジャンプ |
| 差分サマリーパネル | ページごとの差分数・追加/削除/変更の内訳テーブル |
| ズーム同期 | 左右同一倍率でのズーム（個別変更も可能） |
| 差分感度スライダー | 差分検出の閾値調整（低い差分を無視するレベル設定） |
| エクスポートボタン | 差分レポート PDF のエクスポート |

### 主要アクション

- ファイルブラウザから 2 ファイルを「比較対象」に追加 → 自動で差分計算開始
- バージョン履歴から比較バージョン選択（同一ドキュメントの版間比較）
- 差分箇所クリック → コメント追加ダイアログ（注釈エディタと連携）
- 「差分レポート出力」→ PDF/Excel 形式で差分箇所一覧を出力

### 権限

engineer 以上。viewer は差分閲覧のみ可（コメント追加不可）。

## 5. OCR 結果ビューア

### 目的
スキャン PDF から抽出した OCR テキストを確認・修正し、検索可能な文書に変換する。AI による補正結果の確認インターフェース。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 左ペイン: PDF 表示 | 元のスキャン画像を表示。OCR 認識領域をバウンディングボックスでハイライト |
| 右ペイン: テキスト | 抽出テキストをブロック単位で表示。認識信頼度（%）を色で表示 |
| 信頼度フィルター | 信頼度閾値スライダー（低信頼度テキストを強調表示） |
| テキスト修正フィールド | インライン編集。修正前/修正後を並列表示 |
| AI 補正提案 | Claude API による誤認識修正候補を表示（受入/却下） |
| 一括クリア | 手動修正をリセット |
| コピーボタン | 全テキストをクリップボードにコピー |
| 保存ボタン | 修正済み OCR テキストを Backend API へ保存 |
| 進捗バー | OCR 処理中の進捗表示（非同期処理） |

### 主要アクション

- バウンディングボックスクリック → 対応テキストブロックへフォーカス
- テキスト修正 → `PATCH /api/v1/documents/{id}` (ocr_text フィールド更新)
- AI 補正候補「受入」→ 修正テキストに自動反映
- 「全テキストコピー」→ 他システム（積算・施工管理）への転記用

### 権限

engineer 以上。OCR 処理結果の閲覧は全ロール可、修正は engineer 以上。

## 6. 電子印鑑・署名

### 目的
国土交通省の電子納品要領・建設 CAD ガイドラインに準拠した認定電子印鑑を PDF に押印する。印影位置の指定・複数印鑑の管理機能を提供する。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 印鑑選択パネル | 登録済み印鑑一覧（氏名印・丸印・角印・役職印）。プレビュー付き |
| 押印モード | ドラッグ＆ドロップで PDF 上の任意位置に配置 |
| サイズ調整 | 印影の拡大縮小（mm 単位入力） |
| 回転調整 | 印影の角度調整 |
| 印鑑登録ダイアログ | 新しい印鑑（PNG/SVG）の登録。デジタル証明書との紐付け（将来） |
| 押印履歴パネル | 押印者・日時・ページ・座標の一覧 |
| 電子署名情報 | 署名者の証明書情報・タイムスタンプ表示（将来実装） |
| 保存ボタン | 電子印鑑情報を含む PDF 生成・保存 |

### 主要アクション

- 印鑑選択 → PDF 上でクリック/ドロップ → 押印確認ダイアログ
- 「印鑑登録」→ PNG 画像アップロードまたはシステム印鑑から選択
- 押印後「確定」→ Backend API へ押印データ保存、ドキュメントステータス更新
- タイムスタンプ付与（RFC 3161 形式）→ 将来の認定 TSA 連携

### 権限

engineer 以上。役職印・角印は manager / admin のみ使用可。

## 7. AI 検索・要約

### 目的
Claude API を用いた自然言語検索・ページ要約・Q&A 機能を提供する。設計者が膨大な仕様書・図面から必要情報を高速に取得できるようにする。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 検索入力フィールド | 自然言語クエリ入力（例：「3 章の排水設備の仕様」） |
| 検索結果リスト | ファイル名・ページ番号・一致テキスト抜粋・関連度スコア |
| フィルター | プロジェクト・文書種別・日付範囲で絞り込み |
| ページ要約パネル | 選択ページの AI 要約（箇条書き形式） |
| Q&A チャットパネル | 選択ドキュメントに対する質問応答チャット UI |
| ハイライト表示 | 検索結果のキーワードを PDF 上でハイライト |
| 検索履歴 | 過去のクエリ履歴（ローカル保存） |
| ブックマーク | 検索結果のページをブックマーク登録 |

### 主要アクション

- クエリ入力 → `GET /api/v1/documents?q=...` (全文検索) + Claude API 意味検索
- 結果クリック → PDF Viewer でそのページを開く + ハイライト表示
- Q&A 送信 → ドキュメントのコンテキストを含む Claude API 呼び出し
- 「要約」ボタン → 現在ページの OCR テキストを Claude API で要約

### 権限
全ロール。AI 機能の有効/無効は設定画面（admin）で制御。

## 8. 電子納品チェッカー

### 目的
国土交通省の「CAD 製図基準」「電子納品要領」に基づき、PDF 文書の準拠状況を自動チェックする。エラー・警告の一覧表示と自動修正機能を提供する。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| チェック対象選択 | 個別ファイル / フォルダ / プロジェクト全体の選択 |
| 準拠規格選択 | 「電子納品要領（最新版）」「CAD 製図基準」「PDF/A-1b」など |
| チェック実行ボタン | 選択した規格に対してバッチチェック実行 |
| エラー一覧テーブル | エラーレベル（Error / Warning / Info）・ルール ID・説明・対象ファイル・ページ |
| エラー詳細パネル | 選択エラーの詳細説明・修正方法・該当箇所のプレビュー |
| 自動修正ボタン | 対応可能なエラーの自動修正（ファイル名規則・メタデータ修正など） |
| チェックレポート出力 | HTML / PDF / Excel 形式でチェック結果レポートを出力 |
| 進捗バー | バッチチェック中の進捗表示 |

### チェック項目（例）

| カテゴリ | チェック内容 |
|---|---|
| ファイル名規則 | 半角英数字・長さ制限・拡張子 |
| PDF/A 準拠 | PDF/A-1b / PDF/A-2b 形式検証 |
| フォント埋め込み | 全フォントの埋め込み確認 |
| ファイルサイズ | 上限サイズ超過 |
| セキュリティ設定 | パスワード保護・印刷禁止フラグ |
| メタデータ | タイトル・作成者・キーワードの必須項目 |
| ページ向き・サイズ | A1/A3 判定・縦横判定 |

### 主要アクション

- プロジェクト選択 → 「チェック実行」→ Backend API + Claude API で非同期チェック
- エラー行クリック → 対象ファイルを PDF Viewer で開き、問題箇所へスクロール
- 「自動修正」→ 修正前バックアップ作成後、対応ルールを自動適用
- 「レポート出力」→ `GET /api/v1/documents/check-report` で PDF/Excel 取得

### 権限

engineer 以上。自動修正は engineer 以上、レポート出力は全ロール可（閲覧者は出力のみ）。

## 9. オフライン同期パネル

### 目的
ネットワーク未接続の工事現場でもアプリを使用できるよう、ローカルキャッシュの管理と、接続復旧後の同期・競合解決 UI を提供する。

### 主要 UI 要素

| 要素 | 説明 |
|---|---|
| 接続ステータスインジケーター | オンライン / オフライン / 同期中 の状態表示（色分け） |
| キャッシュ一覧 | オフライン利用可能な文書リスト。ダウンロード済みサイズ・最終同期日時 |
| 同期キュー | アップロード待ち・ダウンロード待ちの操作一覧 |
| 競合ファイル一覧 | サーバーとローカルで同時編集が発生したファイルの一覧 |
| 競合解決 UI | 「ローカル版を使用」「サーバー版を使用」「マージ」の選択 |
| 手動同期ボタン | 即時同期の強制実行 |
| キャッシュ設定 | オフライン保存するプロジェクト・フォルダの選択、キャッシュ上限サイズ設定 |
| 同期ログ | 同期処理の成功/失敗履歴 |

### 主要アクション

- 「オフライン利用に追加」→ 選択ファイルをローカルキャッシュにダウンロード
- ネットワーク復旧検知 → 自動でキュー内の操作を Backend API へ送信
- 競合検出時 → 通知でユーザーに解決を促す → 競合解決ダイアログを表示
- 「今すぐ同期」→ デルタ同期（変更差分のみ転送）

### 権限
全ロール。キャッシュ設定は本人の文書のみ制御可。
